# i18n 국제화 계획

## 목표

한국어(기본)와 영어를 지원하는 다국어 처리. URL 구조를 변경하지 않고 유저 설정 및 브라우저 언어 기반으로 로케일을 감지.

## 라이브러리

`next-intl` v3 — Next.js 15 App Router 공식 지원

## 지원 언어

| 코드 | 언어 | 기본값 |
|------|------|--------|
| `ko` | 한국어 | ✅ |
| `en` | 영어 | |

## 로케일 감지 전략

URL prefix 없음 (`localePrefix: 'never'`). 우선순위 순으로:

1. 유저 프로필의 `locale` 필드 (Firebase, 로그인 시)
2. 쿠키 (`NEXT_LOCALE`)
3. `Accept-Language` 헤더 (브라우저 기본 언어)
4. fallback: `ko`

## 파일 구조

```
messages/
  ko.json          # 한국어 메시지 (기본)
  en.json          # 영어 메시지

src/
  i18n/
    routing.ts     # locales, defaultLocale 상수 정의
    request.ts     # next-intl 서버 설정 (getRequestConfig)
  middleware.ts    # next-intl 미들웨어 (locale 감지 → 쿠키 설정)
```

## 메시지 파일 네임스페이스 구조

```json
{
  "common": { "cancel": "취소", "save": "저장", ... },
  "home": { "hero": {...}, "teams": {...}, "logic": {...} },
  "team": { "header": {...}, "sidebar": {...}, "matrix": {...}, "taskForm": {...} },
  "my": { "profile": {...}, "avatar": {...}, "theme": {...}, "teams": {...} },
  "auth": { "login": {...} },
  "join": { "page": {...} }
}
```

## 주차 표기 i18n

`week.ts`의 `formatWeekLabel` 함수는 현재 한국어 하드코딩:
- 한국어: `2025년 4월 3주차`
- 영어: `Week 3, April 2025`

locale 파라미터를 받아 분기 처리하도록 수정. 테스트도 양 언어 케이스 추가.

## 타입 변경 요약

```ts
// src/types/index.ts
type User = {
  // 기존 필드...
  locale?: 'ko' | 'en'
}
```

## 규칙

- 서버 컴포넌트: `getTranslations()` (async)
- 클라이언트 컴포넌트: `useTranslations()` 훅
- 날짜/숫자 포맷: `useFormatter()` 훅
- 메시지 키는 camelCase, 네임스페이스는 페이지/도메인 단위로 분리
